# models

This directory holds model governance, routing, and evaluation artifacts.

## Intended contents
- `registry/` model inventory and approval state
- `routing/` routing definitions and role assignment
- `benchmarks/` evaluation templates and score records
- `templates/` reusable model review templates
- `policies/` model directory, naming, and hygiene standards

## Rule
Every model on disk should have a documented record, status, and reason for approval or rejection.
