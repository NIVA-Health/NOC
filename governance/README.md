# governance

This directory holds the semantic, policy, and safety authority layer for NOC.

## Intended contents
- `ontology/` semantic structure and entity relationships
- `canonical-terms/` naming rules and allowed terms
- `autonomy/` action boundaries and approval tiers
- `contracts/` agent runtime and prompt contracts
- `failure-modes/` failure catalog and escalation rules
- `golden-examples/` expected strong examples
- `evals/` scoring standards and review rubrics
- `queue/` unresolved governance items
- `memory/` retention and memory handling rules
- `taxonomy/` task classes and routing categories

## Rule
Governance files define the rules. Runtime files should conform to them.
